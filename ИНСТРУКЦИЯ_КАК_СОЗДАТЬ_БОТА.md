# Инструкция: как создать Telegram-бота с нуля

## Шаг 1 — Создать бота в BotFather

1. Открой Telegram, найди [@BotFather](https://t.me/BotFather)
2. Напиши `/newbot`
3. BotFather спросит **имя бота** — это отображаемое имя (например: `Копирайтер`)
4. Затем спросит **username** — это адрес бота, должен заканчиваться на `bot` (например: `Alexey_copy_bot`)
5. BotFather выдаст **токен** — длинная строка вида:
   ```
   8114574085:AAGdccNOClMboeij1xGslNk0JhFsb3zZTGk
   ```
   **Сохрани его — это пароль твоего бота.**

---

## Шаг 2 — Создать файл .env с ключами

В папке проекта создай файл `.env` (без расширения, точка в начале):

```
TELEGRAM_BOT_TOKEN=вставь_токен_от_BotFather
OPENROUTER_API_KEY=вставь_ключ_от_OpenRouter
```

**OpenRouter** (если нужен AI):
- Зайди на [openrouter.ai](https://openrouter.ai)
- Зарегистрируйся → Keys → Create Key
- Скопируй ключ вида `sk-or-v1-xxxx...`

## Шаг 3 — Структура любого бота (минимальный шаблон)

Каждый бот состоит из трёх файлов:

```
мой_бот/
├── .env           ← ключи (никому не показывать)
├── bot.py         ← запуск бота
└── handlers.py    ← что бот делает в ответ на сообщения
```

### bot.py — точка входа (копируй каждый раз)

```python
import asyncio
import logging
from aiogram import Bot, Dispatcher
from dotenv import load_dotenv
import os
import handlers  # твои обработчики

load_dotenv()
logging.basicConfig(level=logging.INFO)

async def main():
    bot = Bot(token=os.getenv("TELEGRAM_BOT_TOKEN"))
    dp = Dispatcher()
    dp.include_router(handlers.router)
    await dp.start_polling(bot)

if __name__ == "__main__":
    asyncio.run(main())
```

## Шаг 4 — Добавить кнопки меню при /start

Это самое важное — кнопки внизу экрана (`ReplyKeyboard`).

### Пример handlers.py с кнопками:

```python
from aiogram import Router, F
from aiogram.filters import Command
from aiogram.types import Message, ReplyKeyboardMarkup, KeyboardButton

router = Router()

# Создаём клавиатуру с кнопками
MENU = ReplyKeyboardMarkup(
    keyboard=[
        # Первый ряд — две кнопки рядом
        [KeyboardButton(text="✍️ Кнопка 1"), KeyboardButton(text="📋 Кнопка 2")],
        # Второй ряд — две кнопки рядом
        [KeyboardButton(text="🎨 Кнопка 3"), KeyboardButton(text="⚙️ Кнопка 4")],
        # Третий ряд — одна кнопка на всю ширину
        [KeyboardButton(text="❓ Помощь")],
    ],
    resize_keyboard=True,   # кнопки компактного размера
    persistent=True,        # кнопки всегда видны, не скрываются
)

# Команда /start — показываем приветствие и кнопки
@router.message(Command("start"))
async def cmd_start(message: Message):
    await message.answer(
        "Привет! Я твой бот.\n\nВыбери действие:",
        reply_markup=MENU,  # прикрепляем клавиатуру
    )

# Обработчик нажатия кнопки "✍️ Кнопка 1"
@router.message(F.text == "✍️ Кнопка 1")
async def btn_1(message: Message):
    await message.answer("Ты нажал Кнопку 1!")

# Обработчик нажатия кнопки "📋 Кнопка 2"
@router.message(F.text == "📋 Кнопка 2")
async def btn_2(message: Message):
    await message.answer("Ты нажал Кнопку 2!")

# Обработчик нажатия "❓ Помощь"
@router.message(F.text == "❓ Помощь")
async def btn_help(message: Message):
    await message.answer("Это справка. Нажми любую кнопку меню.")

# Любое другое сообщение
@router.message()
async def handle_any(message: Message):
    await message.answer(f"Получил: {message.text}")
```

## Шаг 7 — Кнопка с диалогом (FSM — двухшаговые кнопки)

Обычная кнопка просто отвечает сразу. Но если кнопка должна **спросить что-то у пользователя**, а потом обработать ответ — нужны состояния (FSM).

**Пример: кнопка «Написать пост»**
1. Пользователь нажимает кнопку
2. Бот спрашивает: «О чём написать пост?»
3. Пользователь пишет тему
4. Бот генерирует пост

Без FSM бот не знает, что следующее сообщение — это тема поста, а не случайный текст.

### Как добавить FSM в handlers.py:

```python
from aiogram import Router, F
from aiogram.types import Message
from aiogram.fsm.context import FSMContext
from aiogram.fsm.state import State, StatesGroup

router = Router()

# Объявляем состояния
class BotStates(StatesGroup):
    waiting_for_topic = State()      # ждём тему поста
    waiting_for_plan = State()       # ждём параметры плана

# Кнопка нажата → спрашиваем тему → запоминаем состояние
@router.message(F.text == "✍️ Написать пост")
async def btn_write_post(message: Message, state: FSMContext):
    await state.set_state(BotStates.waiting_for_topic)  # переходим в состояние
    await message.answer("О чём написать пост? Напиши тему:")

# Пользователь написал тему → обрабатываем
@router.message(BotStates.waiting_for_topic)
async def handle_topic(message: Message, state: FSMContext):
    await state.clear()  # сбрасываем состояние
    topic = message.text
    await message.answer(f"Пишу пост на тему «{topic}»...")
    # здесь вызов AI или любая другая логика

# ВАЖНО: все остальные кнопки тоже должны сбрасывать состояние
@router.message(F.text == "❓ Помощь")
async def btn_help(message: Message, state: FSMContext):
    await state.clear()  # сбрасываем, иначе бот будет путаться
    await message.answer("Это справка.")
```

### Правила FSM которые нужно помнить:

| Правило | Почему |
|---------|--------|
| Каждая кнопка меню должна делать `await state.clear()` | Иначе старое состояние от другой кнопки будет мешать |
| Сначала `await state.clear()`, потом логика | Порядок важен |
| Состояние живёт пока не сбросишь | Пользователь нажал кнопку А, не ответил, нажал Б — Б должна сбросить состояние А |

## Шаг 8 — Порядок роутеров и частые ловушки

### Ловушка 1: catch-all перехватывает все сообщения

Если у тебя есть файловый обработчик — обязательно добавь фильтр `F.document`, иначе он съест все текстовые сообщения:

```python
# ❌ Неправильно — перехватывает ВСЕ сообщения
@router.message()
async def handle_file(message: Message, bot: Bot):
    if not message.document:
        return  # молча проглатывает все текстовые сообщения!

# ✅ Правильно — только файлы
@router.message(F.document)
async def handle_file(message: Message, bot: Bot):
    ...
```

### Ловушка 2: порядок роутеров в bot.py

Роутеры подключаются в порядке приоритета. Правильный порядок:

```python
dp.include_router(commands.router)   # 1. /start, /help и другие команды
dp.include_router(callbacks.router)  # 2. нажатия inline-кнопок
dp.include_router(files.router)      # 3. загрузка файлов
dp.include_router(messages.router)   # 4. текстовые сообщения и кнопки меню
```

Если поставить `messages.router` раньше `files.router` — файлы не будут обрабатываться.

### Ловушка 3: эмодзи должны совпадать точно

```python
# Клавиатура
KeyboardButton(text="✍️ Написать пост")

# Фильтр — должен быть ТОЧНО такой же текст
@router.message(F.text == "✍️ Написать пост")  # ✅ совпадает
@router.message(F.text == "✍ Написать пост")   # ❌ другой эмодзи — не сработает
```

## Правила которые нужно помнить

| Правило | Почему |
|---------|--------|
| Никогда не публикуй `.env` в GitHub | В нём токен — кто угодно получит доступ к боту |
| Всегда добавляй `.env` в `.gitignore` | Защита от случайной публикации |
| Используй `python3`, не `python` | На Mac по умолчанию работает только `python3` |
| Не закрывай терминал пока работает бот | Бот — это процесс, он живёт пока запущен |
| Текст кнопки в коде должен совпадать точно | `F.text == "✍️ Написать пост"` — включая эмодзи и пробелы |

## Частые ошибки и решения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `ModuleNotFoundError` | Библиотека не установлена | `pip3 install aiogram python-dotenv` |
| Бот не отвечает | Процесс не запущен | Запусти `python3 bot.py` в терминале |
| Бот не отвечает (2) | Терминал закрыт | Открой заново и запусти |
| `Invalid token` | Неверный токен в `.env` | Проверь токен от BotFather |
| Кнопки не появляются | Нет `reply_markup=MENU` в `message.answer()` | Добавь параметр |
| Кнопка не реагирует | Текст в коде и на кнопке не совпадает | Проверь точное совпадение включая эмодзи |
| Кнопка нажата, бот молчит | Другой роутер перехватил сообщение | Добавь `F.document` к файловому обработчику |
| После нажатия кнопки А, кнопка Б не работает | FSM-состояние не сбрасывается | Добавь `await state.clear()` в каждый обработчик кнопки |
| Бот принимает тему как неизвестную команду | FSM не активен — пользователь написал без нажатия кнопки | Настрой диспетчер или используй FSM-состояние |
