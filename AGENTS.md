# КопиБот — Архитектура агентов

## Что реализовано сейчас

КопиБот использует **7 AI-агентов**, каждый отвечает за свою задачу. Все агенты работают через OpenRouter (API-прокси к моделям Claude).

```
Пользователь пишет в Telegram
         │
         ▼
  ┌─────────────┐
  │  ДИСПЕТЧЕР  │  ← классифицирует запрос (локально + haiku)
  └──────┬──────┘
         │
    ┌────┴──────────────────────────────┐
    │                                   │
    ▼                                   ▼
Написать пост                    Контент-план
    │                                   │
    ├── Ресерчер (sonnet)          Планировщик (sonnet)
    ├── Генератор (sonnet)
    └── Критик (opus)

Загрузить файл → Аналитик стиля (opus)
Редактировать текст → Редактор (sonnet)
```

---

## Агенты

### 1. Диспетчер (Dispatcher)

**Файл:** `agents/dispatcher.py`
**Модель:** `claude-haiku-4-5` (быстрый и дешёвый — нужен только для классификации)

**Что делает:** Первым обрабатывает каждое сообщение. Определяет намерение пользователя и направляет к нужным агентам.

**Как определяет намерение:**
1. Сначала проверяет по ключевым словам локально (без API — мгновенно)
2. Если не уверен — спрашивает у LLM

**Распознаёт:**

| Намерение | Примеры запроса |
|-----------|----------------|
| `generate_post` | "напиши пост о...", любая тема |
| `edit_post` | "короче [текст]", "живее [текст]" |
| `create_plan` | "создай план на месяц", "продажи, 2 недели" |
| `show_plan` | "покажи план" |
| `analyze_style` | "мой стиль", "стилевой профиль" |
| `critique` | "оцени: [текст]" |
| `next_post` | "следующий пост" |

---

### 2. Аналитик стиля (StyleAnalyst)

**Файл:** `agents/style_analyst.py`
**Модель:** `claude-opus-4-6` (нужно глубокое понимание языка и нюансов)

**Что делает:** Анализирует загруженные посты и создаёт стилевой профиль автора.

**Запускается:** когда пользователь прикрепляет файл `.md` или `.json` с архивом постов.

**Результат сохраняется** в `data/style_guide.json` и используется всеми остальными агентами.

**Что анализирует:**
- Тон и интонацию
- Ритм предложений (короткие/длинные)
- Характерные зачины постов
- Предпочитаемые и запрещённые слова
- Использование эмодзи и хэштегов
- Типичные структурные паттерны

---

### 3. Ресерчер (Researcher)

**Файл:** `agents/researcher.py`
**Модель:** `claude-sonnet-4-6`

**Что делает:** Перед написанием поста собирает факты, углы подачи, идеи по теме. Генератор пишет не из воздуха, а на основе готового брифа.

**Запускается:** автоматически перед каждой генерацией поста.

---

### 4. Генератор (Generator)

**Файл:** `agents/generator.py`
**Модель:** `claude-sonnet-4-6`

**Что делает:** Пишет пост на основе:
- Темы от пользователя
- Брифа от Ресерчера
- Стилевого профиля автора

**Запускается:** после Ресерчера, до Критика.

---

### 5. Критик (Critic)

**Файл:** `agents/critic.py`
**Модель:** `claude-opus-4-6` (нужна точность суждения)

**Что делает:** Оценивает сгенерированный пост по 5 критериям и принимает решение — публиковать, доработать или переписать.

**Критерии оценки (каждый 1–10):**
- Стиль — соответствие авторскому голосу
- Структура — логика и построение
- Зацепка — сила первого предложения
- Фактура — глубина и ценность
- Длина — соответствие формату Telegram

**Решения:**
- `8–10` → публиковать как есть
- `5–7` → отправить Редактору на доработку
- `1–4` → переписать с нуля

**Итоговый ответ пользователю** содержит оценку: `— — —\n_Оценка Критика: 7/10_`

---

### 6. Редактор (Editor)

**Файл:** `agents/editor.py`
**Модель:** `claude-sonnet-4-6`

**Что делает:** Редактирует текст по конкретной команде. Работает как с постами от Генератора, так и с текстами пользователя.

**Команды редактуры:**

| Команда в боте | Что делает |
|----------------|------------|
| `живее [текст]` | Убирает AI-шаблонность, добавляет живость |
| `короче [текст]` | Сжимает без потери смысла |
| `хлестче [текст]` | Делает острее, сильнее финал |
| `проще [текст]` | Упрощает язык |
| `зацепи [текст]` | Переписывает первое предложение |
| `разговорно [текст]` | Переводит в неформальный стиль |
| `экспертно [текст]` | Добавляет авторитетность |
| `реструктурируй [текст]` | Перестраивает логику и абзацы |

---

### 7. Планировщик (ContentPlanner)

**Файл:** `agents/content_planner.py`
**Модель:** `claude-sonnet-4-6`

**Что делает:** Всё связанное с контент-планом:
- Создаёт план по теме и периоду
- Парсит загруженный файл плана
- Форматирует план для показа в Telegram
- Отмечает выполненные пункты

**Хранение:** `data/content_plan.json`

**Формат пункта плана:**
```json
{
  "id": "001",
  "date": "2025-04-01",
  "topic": "Как я нашёл первых клиентов",
  "format": "short",
  "rubric": "личный опыт",
  "status": "planned",
  "notes": "рассказать через конкретный кейс"
}
```

## Полный путь запроса «Написать пост»

```
Пользователь нажал «✍️ Написать пост»
        ↓
Бот спросил: «О чём написать?»
        ↓
Пользователь написал тему
        ↓
Ресерчер собирает фактуру по теме
        ↓
Генератор пишет пост (по фактуре + стилю)
        ↓
Критик оценивает пост
  ├── оценка 8–10 → отправить пользователю
  ├── оценка 5–7 → Редактор дорабатывает → отправить
  └── оценка 1–4 → Генератор пишет заново → отправить
        ↓
Пользователь получает пост + оценку Критика
```

## Структура файлов

```
КопиБот/
├── bot.py                  ← точка входа, запуск бота
├── config.py               ← токены, пути к файлам, названия моделей
├── orchestrator.py         ← координирует агентов, не AI
│
├── agents/
│   ├── base.py             ← базовый класс (один метод call_llm)
│   ├── dispatcher.py       ← классификация запросов
│   ├── style_analyst.py    ← анализ стиля
│   ├── researcher.py       ← фактура по теме
│   ├── generator.py        ← написание поста
│   ├── editor.py           ← редактура
│   ├── critic.py           ← оценка качества
│   └── content_planner.py  ← работа с планом
│
├── handlers/
│   ├── commands.py         ← /start, /план, /стиль, /следующий
│   ├── messages.py         ← кнопки меню + FSM диалоги
│   ├── files.py            ← загрузка .md/.json файлов
│   └── callbacks.py        ← нажатия inline-кнопок (Настройки)
│
├── tools/
│   ├── style_store.py      ← чтение/запись style_guide.json
│   ├── plan_store.py       ← чтение/запись content_plan.json
│   └── file_parser.py      ← парсинг MD и JSON архивов
│
└── data/
    ├── style_guide.json    ← стилевой профиль автора
    └── content_plan.json   ← контент-план
```

## Модели и стоимость (OpenRouter)

| Агент | Модель | Когда используется |
|-------|--------|--------------------|
| Диспетчер | claude-haiku-4-5 | Каждое сообщение (если локальный матч не сработал) |
| Аналитик стиля | claude-opus-4-6 | При загрузке архива |
| Ресерчер | claude-sonnet-4-6 | Перед написанием поста |
| Генератор | claude-sonnet-4-6 | При написании поста |
| Критик | claude-opus-4-6 | После генерации |
| Редактор | claude-sonnet-4-6 | При редактировании |
| Планировщик | claude-sonnet-4-6 | При работе с планом |

**Один пост** = примерно 3–5 API-вызовов (ресерчер + генератор + критик ± редактор).
При балансе $5 на OpenRouter хватит на **500–1000 постов**.
