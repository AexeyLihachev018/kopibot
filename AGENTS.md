# КопиБот — Архитектура агентов

## Обзор системы

Мультиагентная система для Telegram-копирайтера, который пишет в авторском стиле на основе архива канала. Агенты работают в связке через оркестратора — каждый отвечает за свою зону ответственности.

---

## Агенты

### 1. Оркестратор (Orchestrator)

**Роль:** Главный координатор всей системы. Принимает запрос пользователя, определяет какие агенты подключить, в каком порядке, и собирает финальный результат.

**Модель:** `claude-opus-4-6`
_(самая мощная модель — оркестратор должен понимать контекст целиком и принимать верные решения о маршрутизации)_

**Ответственность:**
- Парсинг входящего запроса от пользователя
- Выбор цепочки агентов под конкретную задачу
- Передача контекста между агентами
- Сборка финального ответа
- Обработка ошибок и повторные попытки

**Инструменты:**
- Вызов любого агента из системы
- Чтение/запись в общую память (context store)
- Логирование всех шагов

**Триггеры:**
- Любое сообщение от пользователя в Telegram

---

### 2. Архитектор (Architect)

**Роль:** Проектирует структуру ответа или серии постов до их написания. Думает стратегически — как контент будет выглядеть в целом, какова логика, нарратив, структура.

**Модель:** `claude-opus-4-6`
_(сложное аналитическое мышление, работа со структурами и стратегией)_

**Ответственность:**
- Проектирование структуры поста / серии постов
- Определение нарратива и ключевых месседжей
- Построение контент-плана под заданную тему
- Анализ загруженных файлов канала и выявление паттернов

**Инструменты:**
- Чтение MD/JSON файлов архива канала
- Чтение контент-плана
- Запись структурных схем для Копирайтера

**Входные данные:**
- Тема / запрос пользователя
- Файлы архива (MD, JSON)
- Существующий контент-план (если есть)

**Выходные данные:**
- Структурный план поста или серии
- Список ключевых тезисов

---

### 3. Декомпозер (Task Decomposer)

**Роль:** Разбивает сложные или объёмные задачи на атомарные подзадачи. Работает между Оркестратором и рабочими агентами.

**Модель:** `claude-sonnet-4-6`
_(быстрый, точный — задача механическая, но требует понимания контекста)_

**Ответственность:**
- Декомпозиция задачи на шаги с зависимостями
- Определение приоритетов подзадач
- Назначение каждой подзадачи конкретному агенту
- Оценка завершённости задачи

**Пример работы:**
```
Входящая задача: "Напиши 5 постов по контент-плану в моём стиле"
Декомпозиция:
  1. Архитектор → анализ стиля из архива
  2. Архитектор → анализ контент-плана
  3. Копирайтер × 5 → написание каждого поста
  4. Редактор × 5 → редактура каждого поста
  5. Тестировщик → проверка соответствия стилю
```

---

### 4. Стилист (Style Analyzer)

**Роль:** Глубокий анализ авторского стиля из архива канала. Формирует "стилевой профиль" — набор правил, которым должны следовать все генерирующие агенты.

**Модель:** `claude-sonnet-4-6`

**Ответственность:**
- Анализ лексики, синтаксиса, ритма текстов
- Выявление характерных конструкций и оборотов
- Определение тональности, степени экспрессии
- Составление и обновление Style Guide
- Оценка соответствия нового текста стилю автора (0–100%)

**Поддерживаемые форматы архива:**
- `.md` — выгрузки постов в Markdown
- `.json` — структурированные выгрузки (Telethon, TGStat и др.)

**Выходные данные:**
```json
{
  "tone": "прямой, без воды, иногда саркастичный",
  "sentence_length": "короткие, рубленые",
  "vocabulary": ["слова-маркеры", "характерные обороты"],
  "structure_patterns": ["тезис → пример → вывод"],
  "forbidden": ["канцеляризмы", "пассивный залог", "воды"]
}
```

---

### 5. Планировщик контента (Content Planner)

**Роль:** Работает с контент-планом — читает, создаёт, обновляет. Знает что уже опубликовано, что запланировано, что нужно сгенерировать.

**Модель:** `claude-sonnet-4-6`

**Ответственность:**
- Чтение и парсинг загруженного контент-плана (MD/JSON/CSV)
- Создание нового контент-плана по теме/периоду
- Отметка выполненных задач
- Предложение тем на основе анализа архива и трендов
- Построение тематических рубрик

**Поддерживаемые форматы контент-плана:**
- Markdown таблица
- JSON с полями `date`, `topic`, `format`, `status`
- Произвольный текстовый формат (парсится LLM)

---

### 6. Копирайтер (Copywriter)

**Роль:** Генерирует посты в авторском стиле. Главный "пишущий" агент системы.

**Модель:** `claude-sonnet-4-6`

**Ответственность:**
- Написание поста по теме и структуре от Архитектора
- Следование Style Guide от Стилиста
- Учёт формата (лонгрид, короткий пост, сторис, анонс)
- Генерация нескольких вариантов по запросу
- Интеграция CTA, хэштегов, эмодзи — только если это в стиле автора

**Режимы работы:**
| Режим | Описание |
|-------|----------|
| `generate` | Создать пост с нуля по теме |
| `continue` | Продолжить серию в том же стиле |
| `rephrase` | Переписать чужой текст в авторском стиле |

---

### 7. Редактор (Editor)

**Роль:** Редактирует готовые тексты. Делает посты живее, короче, острее или переформатирует под нужный тон.

**Модель:** `claude-sonnet-4-6`

**Ответственность:**
- Выполнение конкретных редакторских задач по запросу

**Доступные режимы редактуры:**

| Команда | Что делает |
|---------|------------|
| `humanize` | Убирает AI-шаблонность, добавляет живость и неровность |
| `shorten` | Сжимает без потери смысла, убивает воду |
| `punch` | Делает текст хлёстким — острее формулировки, сильнее концовки |
| `simplify` | Упрощает язык, убирает сложные конструкции |
| `tone:casual` | Переводит в разговорный регистр |
| `tone:expert` | Добавляет авторитетность и экспертность |
| `hook` | Переписывает первое предложение — чтобы цепляло |
| `structure` | Перестраивает логику и абзацы |

**Принцип работы:**
- Всегда сверяется со Style Guide от Стилиста
- Не меняет смысл без явного разрешения
- Показывает diff (до/после) по запросу

---

### 8. Тестировщик (QA Agent)

**Роль:** Финальная проверка всего — текстов, работы агентов, соответствия стилю, корректности данных.

**Модель:** `claude-sonnet-4-6`
_(для глубоких проверок) + `claude-haiku-4-5-20251001` (для быстрых автоматических чеков)_

**Ответственность:**
- Проверка сгенерированных постов на соответствие Style Guide
- Валидация структуры контент-плана
- Тестирование цепочек агентов на типовых сценариях
- Проверка корректности парсинга входных файлов
- Выдача score и список замечаний

**Чеклист для поста:**
```
✓ Соответствует тональности автора?
✓ Нет AI-клише и канцеляризмов?
✓ Длина соответствует формату?
✓ Есть сильный первый абзац?
✓ Есть чёткий месседж / вывод?
✓ Хэштеги / эмодзи в норме стиля автора?
✓ Нет фактических противоречий с архивом?
```

## Типовые сценарии работы

### Сценарий 1: Написать пост по теме
```
Пользователь → Оркестратор → Декомпозер
→ Стилист (загружает Style Guide)
→ Архитектор (структура поста)
→ Копирайтер (пишет)
→ Редактор (polish)
→ Тестировщик (проверка)
→ Оркестратор → Пользователь
```

### Сценарий 2: Загрузить архив и настроить стиль
```
Пользователь (файлы MD/JSON) → Оркестратор
→ Стилист (анализ + создание Style Guide)
→ Тестировщик (валидация Style Guide)
→ Оркестратор → Пользователь ("Стиль настроен")
```

### Сценарий 3: Отредактировать готовый пост
```
Пользователь (текст + команда) → Оркестратор
→ Редактор (применяет команду)
→ Тестировщик (сверка со стилем)
→ Оркестратор → Пользователь
```

### Сценарий 4: Создать контент-план на месяц
```
Пользователь → Оркестратор → Декомпозер
→ Стилист (контекст стиля)
→ Архитектор (темы и рубрики)
→ Планировщик (формирует план)
→ Тестировщик (проверка полноты и логики)
→ Оркестратор → Пользователь
```

### Сценарий 5: Написать посты по контент-плану
```
Пользователь (файл плана) → Оркестратор → Декомпозер
→ Планировщик (парсит план)
→ [Для каждого поста]:
   → Архитектор + Копирайтер + Редактор
→ Тестировщик (пакетная проверка)
→ Оркестратор → Пользователь
```

## Внутренние агенты бота (Bot-level Agents)

Второй уровень системы — агенты, живущие непосредственно внутри Telegram-бота. Они работают в реальном времени на каждое сообщение пользователя. Оркестратор верхнего уровня вызывает их как инструменты.

```
Входящее сообщение
        │
        ▼
  ┌─────────────┐
  │  ДИСПЕТЧЕР  │  ◄── первый всегда
  └──────┬──────┘
         │  intent + route
         ▼
┌────────────────────────────────────────────┐
│           Активированные агенты            │
│                                            │
│  Аналитик    Ресерчер    Контент-планер    │
│  стиля                                     │
│                                            │
│  Генератор   Редактор    Критик            │
└────────────────────────────────────────────┘
```

---

### Б-1. Диспетчер (Dispatcher)

**Роль:** Первая точка входа для каждого сообщения. Классифицирует намерение пользователя и определяет какие агенты нужно запустить.

**Модель:** `claude-haiku-4-5-20251001`
_(задача простая и должна выполняться мгновенно — дешевле и быстрее всех)_

**Ответственность:**
- Классификация интента пользователя
- Роутинг к нужной цепочке агентов
- Извлечение параметров из сообщения (тема, формат, команда)
- Определение режима работы

**Классификатор интентов:**

| Intent | Триггер | Цепочка агентов |
|--------|---------|-----------------|
| `upload_archive` | загружен MD/JSON файл | Аналитик стиля |
| `upload_plan` | загружен контент-план | Контент-планер |
| `generate_post` | "напиши пост", "создай", тема | Ресерчер → Генератор → Критик |
| `edit_post` | "сделай короче", "хлестче", + текст | Редактор → Критик |
| `create_plan` | "план на месяц", "контент-план" | Контент-планер |
| `generate_from_plan` | "пиши по плану" | Контент-планер → Генератор |
| `analyze_style` | "какой у меня стиль", "обнови стиль" | Аналитик стиля |
| `critique` | "оцени этот текст", "проверь" | Критик |
| `research` | "найди фактуру", "что известно о..." | Ресерчер |

**Выходные данные:**
```json
{
  "intent": "generate_post",
  "topic": "как я перестал откладывать решения",
  "format": "short",
  "params": {},
  "route": ["researcher", "generator", "critic"]
}
```

---

### Б-2. Аналитик стиля (Style Analyst)

**Роль:** Извлекает стилевой профиль из загруженных файлов архива канала. Создаёт и обновляет живой Style Guide, которым пользуются Генератор и Редактор.

**Модель:** `claude-opus-4-6`
_(тонкий анализ языка, паттернов, интонации — требует глубокого понимания текста)_

**Ответственность:**
- Парсинг и нормализация MD и JSON файлов архива
- Глубокий лингвистический анализ текстов
- Выявление авторских маркеров: лексика, ритм, структуры, интонация
- Создание / инкрементальное обновление Style Guide
- Версионирование профиля при обновлении архива

**Поддерживаемые форматы:**
- `.md` — посты в Markdown (выгрузка вручную)
- `.json` — структурированная выгрузка (Telethon, TGStat, экспорт Telegram Desktop)

**Структура Style Guide (выход):**
```json
{
  "version": "2024-03-22",
  "tone": "прямой, саркастичный, без воды",
  "sentence_rhythm": "короткие + средние, без длинных периодов",
  "typical_openers": ["Вот правда:", "Никто не говорит, но..."],
  "vocabulary": {
    "preferred": ["конкретно", "факт", "решение"],
    "forbidden": ["в рамках", "осуществлять", "является"]
  },
  "structure_patterns": ["провокация → факт → вывод"],
  "avg_post_length": 320,
  "emoji_usage": "редко, только по смыслу",
  "hashtag_usage": false,
  "signature_moves": ["риторический вопрос в конце", "цифра в первом предложении"]
}
```

---

### Б-3. Ресерчер (Researcher)

**Роль:** Готовит фактуру по теме перед написанием поста. Собирает факты, примеры, цифры — чтобы Генератор писал не из головы, а из материала.

**Модель:** `claude-sonnet-4-6`
_(требует рассуждения и синтеза информации, но не на уровне стратегии)_

**Ответственность:**
- Поиск релевантных фактов, статистики, кейсов по теме
- Анализ уже опубликованных постов автора по этой теме (из архива)
- Проверка на повторяемость — не предлагать то, о чём уже писали
- Формирование структурированного брифа для Генератора

**Источники:**
- Архив постов канала (приоритет — свои материалы)
- Веб-поиск (если подключён инструмент)
- Знания модели (с указанием что это LLM-knowledge, не факт)

**Выходные данные — бриф для Генератора:**
```markdown
## Бриф по теме: "прокрастинация у основателей"

**Ключевые факты:**
- 92% основателей откладывают решения о найме (источник: LLM)
- Среднее время от "надо уволить" до увольнения — 6 месяцев

**Из архива канала:**
- 12.01.2024 — писал про принятие решений, угол был про скорость
- Тема прокрастинации напрямую не раскрывалась

**Возможные углы:**
1. Личная история + вывод
2. Цифра в лоб → почему так → что делать

**Чего избегать:**
- Советы в духе "просто начни" — уже было у других
```

---

### Б-4. Генератор (Generator)

**Роль:** Пишет посты на основе брифа от Ресерчера и Style Guide от Аналитика стиля. Главный "пишущий" агент бота.

**Модель:** `claude-sonnet-4-6`

**Ответственность:**
- Написание поста строго по Style Guide
- Работа с брифом от Ресерчера (факты, углы)
- Генерация 1–3 вариантов по запросу
- Подбор формата (короткий / лонгрид / серия)
- Самопроверка перед выдачей: нет ли нарушений стиля

**Параметры запуска:**
```
topic: str          — тема поста
brief: dict         — бриф от Ресерчера
style_guide: dict   — профиль от Аналитика стиля
format: short|long|series
variants: 1|2|3
```

---

### Б-5. Редактор (Editor)

**Роль:** Принимает готовый текст и выполняет конкретную редакторскую команду. Работает как с постами от Генератора, так и с текстами, которые пользователь пишет сам.

**Модель:** `claude-sonnet-4-6`

**Ответственность:**
- Точное выполнение команды редактуры
- Сохранение авторского смысла
- Соблюдение Style Guide при переработке

**Команды редактуры:**

| Команда | Описание |
|---------|----------|
| `humanize` | Убирает AI-шаблонность, добавляет живые детали и неровности |
| `shorten` | Сжимает текст без потери смысла, режет воду |
| `punch` | Делает хлёстким — острее формулировки, сильнее финал |
| `simplify` | Убирает сложные конструкции, упрощает язык |
| `hook` | Переписывает только первое предложение — чтобы зацепило |
| `casual` | Переводит в разговорный регистр |
| `expert` | Добавляет авторитетность и экспертный тон |
| `restructure` | Перестраивает логику и порядок абзацев |
| `cut_half` | Агрессивное сжатие — уполовинить текст |

**Всегда показывает:** оригинал → результат + 1 строка о том, что изменилось и почему.

---

### Б-6. Критик (Critic)

**Роль:** Независимый оценщик качества текста. Ставит оценку от 1 до 10 и даёт конкретные предложения по улучшению. Не переписывает — только судит.

**Модель:** `claude-opus-4-6`
_(оценка качества требует тонкого суждения, не допускает шаблонных ответов)_

**Ответственность:**
- Оценка текста по шкале 1–10 с детализацией по критериям
- Указание конкретных проблемных мест (не "текст слабый", а "третий абзац теряет нить")
- Проверка на соответствие Style Guide
- Решение: публиковать / доработать / переписать
- Опциональная передача на доработку Редактору или Генератору

**Структура оценки:**

```
ОЦЕНКА: 7/10

По критериям:
  Стиль автора:     8/10  — тон выдержан, есть характерные обороты
  Структура:        6/10  — финал провисает, нет точки
  Зацепка (хук):    7/10  — первое предложение работает
  Фактура:          8/10  — есть конкретика, нет воды
  Длина/формат:     6/10  — на 40 слов длиннее оптимума

Проблемы:
  → Абзац 3: теряется нить, можно вырезать
  → Финал: заканчивается вопросом — автор обычно так не делает

РЕШЕНИЕ: доработать
  → Передать Редактору: команды [shorten, punch]
```

**Пороги решений:**
- `1–4` → переписать (отправить Генератору)
- `5–7` → доработать (отправить Редактору с командами)
- `8–10` → публиковать

---

### Б-7. Контент-планер (Content Planner)

**Роль:** Всё что связано с контент-планом. Читает загруженные планы, создаёт новые, отмечает выполненные задачи, предлагает темы.

**Модель:** `claude-sonnet-4-6`

**Ответственность:**
- Парсинг контент-планов в любом формате (MD таблица, JSON, свободный текст)
- Генерация нового плана на период с учётом архива (не повторяться)
- Ведение статуса: `planned → in_progress → done`
- Предложение рубрик и тематических серий
- Экспорт плана в MD или JSON

**Поддерживаемые форматы входа:**
- Markdown таблица
- JSON (`date`, `topic`, `format`, `status`, `notes`)
- Произвольный текст (парсится моделью)

**Формат хранения плана:**
```json
[
  {
    "id": "001",
    "date": "2024-03-25",
    "topic": "Почему основатели боятся нанимать сильных",
    "format": "short",
    "rubric": "управление",
    "status": "planned",
    "brief_ready": false,
    "post_ready": false,
    "notes": ""
  }
]
```

**Команды:**
- `/план` — показать текущий план
- `/план создать [период] [тема/рубрика]` — сгенерировать новый
- `/план готово [id]` — отметить пост опубликованным
- `/план следующий` — что писать по плану сегодня
